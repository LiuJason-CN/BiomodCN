Sequence Design Program

It took us more than 3 weeks to design sequences of Olympic-ring. Many factors needed to be taken into consideration i.e. mismatch, GC ratio, unexpected secondary structure etc. We realized that it is urgent to develop a computer program to simplify the process. Yang Xu came up with an algorithm, called 345 algorithm. The aim of 345 algorithm is to generate a set of DNA strands which could form specific secondary structure following the instruction of users. Each strand could only hybridize with its complementary strand. No mismatch and unexpected self-secondary structure is allowed. Beside duplex region, ploy-T sequences were inserted to avoid unexpected result.

The program is based on a simple rule: any three-base-long oligodeoxynucleotides (ODN) on strand A should not be fully complement with any three-based-long ODN on the other strands. To achieve that, a library containing 32 three-based-long ODNs (4^3/2) were built. Each component ODN cannot fully pair with any other ODNs in the library. Using this strategy, we created four-based-long ODN and five-based-long ODN as well. The algorithm will use these libraries as the fundamental element to generate the ideal DNA sequence.
